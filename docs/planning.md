# 게임 기획 플래닝

## 1. 핵심 컨셉 (한 줄 소개)
- 한 줄 소개: "우유에 과일을 순서대로 넣어 레시피를 완성하는 스무디 제작 게임"
- 키워드 3개: 순서, 레시피, 확장

## 2. 핵심 루프 (30초~3분 반복)
1) 과일 선택 (딸기/바나나 등) →
2) 우유에 순서대로 투입 →
3) 결과 스무디 확인 (정답/오답 피드백) →
4) 레시피 확장 또는 다음 주문 진행

## 3. 플레이어 역할
- 플레이어는 무엇을 조작하는가?: 과일 선택과 투입 순서
- 즉시 이해되는 목표는?: 주문한 스무디 레시피를 정확히 완성

## 3-1. 재료 꺼내기 규칙
- 완성된 스무디에서 재료를 꺼낼 수 있음
- 꺼낸 재료는 "[스무디 이름] [재료 이름]"로 정의
  - 예: 딸기 스무디에서 딸기 꺼내기 → "딸기 스무디 딸기"
  - 예: 바나나 스무디에서 바나나 꺼내기 → "바나나 스무디 바나나"
  - 예: 바나나 딸기 스무디에서 딸기 꺼내기 → "바나나 딸기 스무디 딸기"

## 4. 승패/종료 조건
- 승리/성공 조건: 주문 레시피를 맞히면 성공
- 실패/종료 조건: 잘못된 순서/재료로 오답 처리

## 4-1. 주문 방식
- 제시되는 레시피를 그대로 만드는 것이 기본 규칙
- 레시피 제시 방식: 텍스트

## 4-2. 플레이어 액션
1) 통을 꺼낸다
2) 우유를 넣는다
3) 집는다 → 딸기
4) 집는다 → 바나나
5) 집는다 → 스무디(통째)
6) 놓는다 → 손 슬롯 N
7) 음료를 만든다
8) 음료를 폐기처리 한다
9) 제시하기 (완성한 음료/손에 든 것 제출)

## 4-3. 액션 제약/상태 규칙
- 액션은 되돌릴 수 없음 (취소 불가)
- 딸기/바나나는 우유가 들어 있는 통에만 추가 가능
- 통에는 바나나 최대 1개, 딸기 최대 1개만 들어갈 수 있음
- 테이블에는 동시에 통을 최대 5개까지 꺼낼 수 있음

## 4-4. 음료 만들기 이후
- "음료를 만든다"를 실행하면 스무디가 됨
- 스무디 상태에서 딸기 또는 바나나를 꺼낼 수 있음
- 꺼내기를 수행한 스무디는 통에 남아 있음
- 해당 통은 폐기처리로 정리해야 함
- 스무디에서 재료를 꺼내면 해당 통은 "스무디 이었던것" 상태로 변경
- "스무디 이었던것"은 폐기처리 외에는 다른 동작 불가

## 4-5. 판정 방식
- 제시하기는 새로운 액션
- 제시하기 전까지는 무한히 액션 가능
- 제시하기 이후에는 제시어(레시피)가 변경됨
- 제시하기를 통해 만들어둔 음료가 레시피와 일치하는지 확인
- 제시하기 시 손에 있는 재료도 제출할 수 있음
- 미션 제시어는 반드시 "스무디"로 끝날 필요 없음

## 4-6. 꺼내기 결과
- 꺼내기 행동을 하면 과일에 맞는 수식어가 달린 재료가 손에 들어옴
- 손에 든 재료는 다른 음료에 넣을 수 있음
  - 예: 딸기 스무디 만들기 → 딸기 꺼내기 → 손: "딸기 스무디 딸기"
  - 예: 우유에 "딸기 스무디 딸기"를 넣고 만들기 → "딸기 스무디 딸기 스무디"
- 손에는 동시에 재료를 최대 5개까지 들 수 있음
- 수식어 재료도 일반 재료와 동일하게 통에 넣을 수 있음
- 손에 든 재료는 통에 다시 넣을 수 있음

## 4-7. 폐기 및 통 회수
- 폐기처리하면 해당 통도 함께 사라짐 (테이블의 통 개수 감소)
- 스무디를 통째로 손으로 가지고 오는 액션이 존재
- 스무디를 통째로 손에 넣으면 통도 함께 회수되어 테이블 통 개수가 감소

## 5. 핵심 재미 가설
- 왜 재미있을까?: 순서 규칙을 맞히는 퍼즐/기억 재미
- 1분 안에 느껴야 하는 재미: 맞추면 즉시 결과가 바뀌는 명확한 피드백

## 6. 최소 구현 범위 (MVP)
- 반드시 있어야 하는 것 3가지:
  1) 우유 컵과 과일 선택 UI
  2) 순서 입력 기록 및 결과 판정
  3) 결과 스무디 이름 표시 (딸기/바나나/딸기-바나나/바나나-딸기)

## 7. 다음에 정할 것
- 조작 방식 (키/터치/드래그)
- 난이도 곡선 (초반/중반/후반)
- 아트 방향 (2D/3D, 분위기)

## 8. 입력 방식 (키보드)
- 키보드로 테이블에 올려진 통을 이동/선택
- 스페이스바로 액션 선택/확정
