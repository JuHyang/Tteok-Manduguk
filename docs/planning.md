# 게임 기획 플래닝

## 1. 핵심 컨셉 (한 줄 소개)

- 한 줄 소개: "국물에 떡/만두를 순서대로 넣어 레시피를 완성하는 국 요리 게임"
- 키워드 3개: 순서, 레시피, 확장

## 2. 핵심 루프 (30초~3분 반복)

1. 재료 선택 (떡/만두) →
2. 국물에 순서대로 투입 →
3. 결과 국 확인 (정답/오답 피드백) →
4. 레시피 확장 또는 다음 주문 진행

## 3. 플레이어 역할

- 플레이어는 무엇을 조작하는가?: 재료 선택과 투입 순서
- 즉시 이해되는 목표는?: 주문한 국 레시피를 정확히 완성

## 3-1. 재료 꺼내기 규칙

- 국에서 재료를 꺼낼 수 있음
- 꺼낸 재료는 "[국 이름][재료 이름]"로 정의 (공백 없이)
  - 예: 떡국에서 떡 꺼내기 → "떡국떡"
  - 예: 만두국에서 만두 꺼내기 → "만두국만두"
  - 예: 만두떡국에서 떡 꺼내기 → "만두떡국떡"

## 4. 승패/종료 조건

- 승리/성공 조건: 주문 레시피를 맞히면 성공
- 실패/종료 조건: 잘못된 순서/재료로 오답 처리

## 4-1. 주문 방식

- 제시되는 레시피를 그대로 만드는 것이 기본 규칙
- 레시피 제시 방식: 텍스트

## 4-2. 플레이어 액션

1. 그릇을 꺼낸다 (손 슬롯)
2. 국을 꺼낸다 (손 슬롯)
3. 떡 가져오기 (손 슬롯)
4. 만두 가져오기 (손 슬롯)
5. 그릇에서 떡 꺼내기
6. 그릇에서 만두 꺼내기
7. 국 그릇 집기
8. 놓는다 → 손 슬롯 → 테이블 슬롯
9. 버리기 → 선택 손 슬롯 버리기
10. 제시하기 (완성한 국/손에 든 것 제출)

## 4-3. 액션 제약/상태 규칙

- 액션은 되돌릴 수 없음 (취소 불가)
- 떡/만두는 국이 들어 있는 그릇에만 추가 가능
- 그릇에는 만두 최대 1개, 떡 최대 1개만 들어갈 수 있음
- 테이블에는 동시에 그릇을 최대 5개까지 둘 수 있음

## 4-4. 국 상태

- 국이 들어간 그릇에서 떡 또는 만두를 꺼낼 수 있음
- 꺼내기를 수행한 그릇은 테이블에 남아 있음
- 해당 그릇은 "국 이었던것" 상태로 변경
- "국 이었던것"은 떡/만두 꺼내기 불가, 국 그릇 집기로 손에 가져온 뒤 버리기 가능

## 4-5. 판정 방식

- 제시하기는 새로운 액션
- 제시하기 전까지는 무한히 액션 가능
- 제시하기 이후에는 제시어(레시피)가 변경됨
- 제시하기를 통해 만들어둔 국이 레시피와 일치하는지 확인
- 제시하기 시 손에 있는 재료도 제출할 수 있음
- 미션 제시어는 반드시 "국"으로 끝날 필요 없음 (예: 떡국떡)

## 4-6. 꺼내기 결과

- 꺼내기 행동을 하면 재료에 맞는 수식어가 달린 재료가 손에 들어옴
- 손에 든 재료는 다른 국에 넣을 수 있음
  - 예: 떡국 만들기 → 떡 꺼내기 → 손: "떡국떡"
  - 예: 국에 "떡국떡"를 넣고 만들기 → "떡국떡국"
- 손에는 동시에 재료를 최대 5개까지 들 수 있음
- 수식어 재료도 일반 재료와 동일하게 그릇에 넣을 수 있음
- 손에 든 재료는 그릇에 다시 넣을 수 있음

## 4-7. 버리기 및 그릇 회수

- 버리기는 손 슬롯의 아이템을 제거함
- 국 그릇을 손으로 가지고 오는 액션이 존재
- 국 그릇을 손에 넣으면 테이블 그릇 개수가 감소

## 5. 핵심 재미 가설

- 왜 재미있을까?: 순서 규칙을 맞히는 퍼즐/기억 재미
- 1분 안에 느껴야 하는 재미: 맞추면 즉시 결과가 바뀌는 명확한 피드백

## 6. 최소 구현 범위 (MVP)

- 반드시 있어야 하는 것 3가지:
  1. 국 그릇과 재료 선택 UI
  2. 순서 입력 기록 및 결과 판정
  3. 결과 국 이름 표시 (떡국/만두국/떡만두국/만두떡국)

## 7. 다음에 정할 것

- 조작 방식 (키/터치/드래그)
- 난이도 곡선 (초반/중반/후반)
- 아트 방향 (2D/3D, 분위기)

## 8. 입력 방식 (키보드)

- 키보드로 테이블에 올려진 그릇을 이동/선택 (←→)
- 숫자 1~5로 손 슬롯 선택
- ↑↓로 액션 선택, 스페이스바로 실행
